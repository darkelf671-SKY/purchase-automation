## 2. 전문가 분석

### 2.1 UI/UX 디자이너 관점

**문제점**:
- 같은 구매 데이터를 보여주는데 탭마다 항목 구성·순서·라벨이 다름
- 사용자가 검수탭에서 보던 정보를 이력탭에서 찾으려면 혼란
- "비고" vs "기안비고" — 같은 데이터인데 라벨이 다름

**원칙**: 동일 데이터 = 동일 표시. 탭의 **역할에 따라 추가 항목만 다르게**.

### 2.2 실무 담당자 관점

**검수 업무 시 필요한 정보 (우선순위)**:
1. 기안제목 — 어떤 건인지 식별
2. 품명/규격/수량 — 검수 대상 확인
3. 구매처(선정업체) — 납품자 확인 (현재 **누락**)
4. 금액 — 계약금액 확인 (현재 **누락**)
5. 기안내용 — 참고용
6. 부서명 — 어느 부서 건인지

**이력 조회 시 필요한 정보 (우선순위)**:
1. 기안제목 — 식별
2. 견적1/2 비교 — 가격 비교 확인
3. 기안내용 — 무엇을 구매했는지 확인 (현재 **누락**)
4. 부서명/규격/수량 — 상세 확인
5. 비고/검수메모 — 기록 확인

### 2.3 프론트엔드 아키텍트 관점

**현재 구현 문제**:
- 두 탭이 독립적으로 개발되어 공통 컴포넌트 없음
- `_on_select()` 메서드가 각각 다른 로직으로 데이터 표시
- 다중 품목 표시 형식이 다름:
  - 검수탭: `"토너검정, 토너빨강"` (이름만 나열)
  - 이력탭: `"토너검정(2EA), 토너빨강(1EA)"` (수량 포함)

**개선 방향**:
- 공통 정보 표시 순서 통일
- 다중 품목 요약 형식 통일
- 라벨명 통일

### 2.4 CTO 관점

**리스크 분석**:
- 이력탭에 기안내용 추가 → 상세 패널 높이 증가 → 목록 공간 축소 가능
- 검수탭에 견적 정보 추가 → 입력 영역 시작점이 내려감 → 스크롤 필요 가능
- 두 탭의 역할이 다르므로 **완전 동일**보다는 **공통 부분 통일 + 탭별 추가 항목** 방식

**결정**:
- 공통 기본 정보 블록 통일 (기안제목, 부서, 품명, 규격, 수량)
- 탭별 추가 블록 유지 (검수탭: 입력필드, 이력탭: 견적 비교/URL)
- 기안내용은 길이가 가변적 → 두 탭 모두 포함하되 말줄임 처리

### 2.5 QA 관점

**테스트 매트릭스**:
- 단일 품목 선택 시 양 탭 표시 일치
- 다중 품목 선택 시 양 탭 요약 형식 일치
- 기안내용 없는 건 표시
- 부서명 없는 건 표시
- 검수 완료/미완료 건 표시

### 2.6 보안 관점
- 상세 정보는 읽기 전용 라벨 → XSS/인젝션 위험 없음
- 별도 보안 이슈 없음

## 4. 변경 상세

### 4.1 검수입력탭 변경

| 변경 | 내용 |
|------|------|
| 추가 | 부서명 (기안제목 다음 행, 품명/수량과 같은 줄) |
| 추가 | 구매처·금액 행 (구분선 후, 검수 입력 필드 전) |
| 변경 | "비고" → "기안비고" (라벨 통일) |
| 변경 | "구매품명" → "품명" (라벨 통일) |
| 변경 | "구매수량" → 수량 (부서/품명과 같은 줄로 이동) |
| 변경 | 다중 품목 → `품명(수량단위)` 형식 통일 |
| 유지 | 기안내용 200자 말줄임 |

### 4.2 이력조회탭 변경

| 변경 | 내용 |
|------|------|
| 추가 | 기안내용 행 (규격 다음, 200자 말줄임) |
| 추가 | 품명 표시 (부서명과 같은 줄) |
| 변경 | "기안비고" 위치 → 기안내용 바로 아래 (견적 블록 전) |
| 유지 | 견적1/2 블록 (이력탭 전용) |
| 유지 | 검수메모 (이력탭 전용) |

### 4.3 다중 품목 통일 형식

```
단일: "토너검정"
다중: "토너검정(2EA), 토너빨강(1EA)"  ← 두 탭 모두 이 형식
```

## 5. 리스크 분석

| 리스크 | 확률 | 영향 | 대응 |
|--------|------|------|------|
| 검수탭 정보 영역 확대 → 입력 필드 밀림 | Medium | Low | grid 레이아웃으로 컴팩트 배치 |
| 이력탭 상세 패널 높이 증가 | Medium | Low | 기안내용 1줄 말줄임으로 최소 높이 유지 |
| 기존 _on_select 로직 변경 범위 | Low | Low | 추가만 있고 삭제 없음 |
| 검수 문서 생성 로직 영향 | None | None | 표시 전용, 데이터 흐름 무관 |
