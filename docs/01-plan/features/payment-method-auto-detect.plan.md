# 결제방법 자동 추천 기획 문서

> **Summary**: 업체 관리에서 결제방법 라디오버튼을 제거하고 은행 정보만 등록(가능 수단). 구매 조사 탭에서는 결제방법 선택을 유지하되, 업체의 은행 정보 유무로 기본값을 자동 추천
>
> **Project**: 구매기안 자동화 시스템 v1.0
> **Version**: v1.3 (v1.3-rev2: 구매건별 결제방법 선택 유지)
> **Author**: 전산팀 장길섭
> **Date**: 2026-03-13
> **Status**: Draft
> **Review Team**: CTO + 프론트엔드 아키텍트 + DB 아키텍트 + 보안 전문가 + QA 전략가 + 제품 관리자(PM)

## Executive Summary

| 관점 | 내용 |
|------|------|
| **문제** | 업체 등록 시 결제방법을 수동 선택해야 하는 이중 작업 + Excel 양식에 영문 코드(card/transfer) 노출. 단, 같은 업체라도 구매 건별로 카드/무통장/자동이체가 달라질 수 있음 |
| **해결책** | 업체 관리에서 결제방법 라디오 제거 → 은행 정보만 등록 (= "이 업체는 무통장 가능"). 구매 조사 탭에서 건별 결제방법 선택은 **유지**하되, 업체 은행 정보 기반으로 **기본값 자동 추천** |
| **기능/UX 효과** | 업체 등록 단순화(라디오 제거). Excel 양식 7열 간소화. 구매 시 업체 선택하면 은행 정보 유무에 따라 결제방법 자동 선택되지만, 사용자가 언제든 변경 가능 |
| **핵심 가치** | 업체 등록 UX 단순화 + 영문 코드 노출 제거 + 구매 건별 결제방법 유연성 100% 유지 |

## 핵심 설계 원칙: Capability vs Decision

```
┌──────────────────────────────────────────────────────────────┐
│  업체 관리 (tab_vendor)  = Capability (가능 수단 등록)       │
│  → 은행 정보 있으면 "이 업체는 무통장입금 가능"              │
│  → 은행 정보 없으면 "이 업체는 카드 전용"                    │
│  → 결제방법 라디오버튼 불필요 (은행 정보 자체가 능력)        │
│                                                              │
│  구매 조사 (tab_purchase) = Decision (건별 결정)             │
│  → 업체 선택 시 기본 추천 (은행 있으면 "무통장" 기본 선택)   │
│  → 사용자가 건별로 다른 결제방법 선택 가능                   │
│  → 결제방법 라디오버튼 유지                                  │
└──────────────────────────────────────────────────────────────┘
```

**실무 시나리오**:
| # | 시나리오 | 업체 은행 정보 | 기본 추천 | 사용자 변경 |
|---|---------|-------------|----------|-----------|
| A | 두인시스템 토너 10만원 | 은행 있음 | 무통장입금 | → 법인카드로 변경 |
| B | 두인시스템 서버 500만원 | 은행 있음 | 무통장입금 | 그대로 사용 |
| C | 학교장터 구매 | 은행 없음 | 법인카드 | 그대로 사용 |
| D | 월정액 유지보수 업체 | 은행 있음 | 무통장입금 | → 자동이체로 변경 |

## CTO팀 전문가 브레인스토밍 (6명)

### 1. CTO (기술 리더)

**핵심 판단**: 업체 정보의 결제방법은 "가능 수단(capability)"이지 "확정(decision)"이 아님.
- 설계 원칙: **Capability ≠ Decision**
  - 업체 관리: 은행 정보 = 무통장입금 "가능 여부" 등록
  - 구매 조사: 결제방법 = 이번 건의 "실제 결제" 선택
- `auto_transfer`(자동이체납부) 처리: 별도 체크박스로 분리하여 업체 속성으로 관리
- `_pay_method_var`(구매탭 결제방법): 업체 선택 시 자동 추천, 사용자 변경 허용

**결론**: 업체 관리에서 라디오 제거 + 구매 조사에서 라디오 유지 + 업체 은행 정보 기반 자동 추천

### 2. 프론트엔드 아키텍트

**UI 변경 설계**:

#### 2.1 업체 관리 다이얼로그 (VendorDialog)
- **제거**: 결제방법 라디오버튼 3개
- **유지**: 은행명/예금주/계좌번호 필드 (항상 표시, 항상 입력 가능)
- **추가**: "자동이체납부 업체" 체크박스 1개
- **추가**: 결제방법 자동 감지 미리보기 라벨 ("→ 무통장입금 가능")

#### 2.2 업체 목록 Treeview
- "결제방법" 컬럼 유지 (자동 감지된 기본값 표시)

#### 2.3 구매 조사 탭 (tab_purchase.py) — ★ 라디오 유지
- **유지**: 결제방법 라디오버튼 3개 (법인카드/무통장입금/자동이체납부)
- **변경**: 업체 선택 시 은행 정보 기반으로 기본값 **자동 선택** (추천)
  - 은행 정보 있으면 → "무통장입금" 자동 선택
  - 은행 정보 없으면 → "법인카드" 자동 선택
  - is_auto_transfer=1 → "자동이체납부" 자동 선택
- **유지**: 사용자가 언제든 다른 결제방법으로 변경 가능
- **유지**: 은행 정보 프레임 — "무통장입금" 선택 시 표시

#### 2.4 Excel 양식
- **제거**: "결제방법" 열 (5번째 컬럼)
- **결과**: 헤더 7열 — 상호*, 대표자, 사업자등록번호, 주소, 은행명, 예금주, 계좌번호
- **설명 시트**: 자동 감지 규칙 설명

#### 2.5 일괄 업로드 미리보기 (BulkUploadPreviewDialog)
- "결제방법" 컬럼에 자동 감지된 기본값 표시

### 3. DB 아키텍트

**데이터 모델 전략**:

#### 3.1 vendors 테이블 — payment_method의 역할 변경
- `payment_method` = **기본 추천 결제방법** (업체의 default, 구매 시 자동 선택에 사용)
- 저장 시 은행 정보 기반 자동 계산 (derive)
- `is_auto_transfer` 컬럼 추가 (자동이체 플래그)

#### 3.2 purchases 테이블 — 건별 결제방법 스냅샷
- `payment_method` = **이 건의 확정 결제방법** (구매 시점 사용자 선택 스냅샷)
- 기존 로직 변경 없음

#### 3.3 마이그레이션
```sql
ALTER TABLE vendors ADD COLUMN is_auto_transfer INTEGER DEFAULT 0;
UPDATE vendors SET is_auto_transfer = 1 WHERE payment_method = 'auto_transfer';
```

### 4. 보안 전문가

- 데이터 무결성: `derive_payment_method()` 단일 함수로 일원화
- 기존 데이터: `is_auto_transfer` 플래그 추가만, 기존 값 보존
- 감사 추적: 구매 건의 결제방법은 purchases 테이블에 스냅샷으로 영구 보존

### 6. 제품 관리자 (PM)

#### 6.1 핵심 사용자 스토리
- **US-01**: 담당자가 업체를 등록할 때, 은행 정보만 입력하면 기본 결제방법이 자동 결정되어 별도 선택이 불필요하다
- **US-02**: 담당자가 Excel로 업체를 일괄 등록할 때, 결제방법 열 없이 은행 정보만 넣으면 자동 감지된다
- **US-03**: 담당자가 구매 조사 시 업체를 선택하면, 기본 결제방법이 자동 추천되지만 **이번 건에서 다른 방법으로 변경할 수 있다**
- **US-04**: 기존 구매 이력의 결제방법 정보는 변경되지 않는다

#### 6.2 비즈니스 규칙
- **BR-01**: 업체 등록: 은행명+계좌번호 → 기본값 "무통장입금" 자동 저장
- **BR-02**: 업체 등록: 은행 정보 없음 → 기본값 "법인카드" 자동 저장
- **BR-03**: 업체 등록: is_auto_transfer → 기본값 "자동이체납부" 자동 저장
- **BR-04**: 구매 조사: 업체 선택 시 기본값을 라디오에 자동 선택 (추천)
- **BR-05**: 구매 조사: 사용자가 라디오를 변경하면 그 값이 구매건 결제방법으로 확정
- **BR-06**: HWP 문서: 구매건의 확정 결제방법이 문서에 반영
- **BR-07**: 불완전 은행 정보(한쪽만 입력) → 경고 메시지 + "법인카드" 기본값
